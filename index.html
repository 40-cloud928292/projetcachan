Configuration et tests de notre robot Pololu 3Pi+

Présentation du projet

On est une équipe de trois étudiants en BUT GEII à l'IUT de Vélizy. Ce site sert à présenter notre projet et notre préparation pour le concours de robotique de Cachan.

On utilise un robot Pololu 3Pi+. Notre travail consiste principalement à calibrer les capteurs et à écrire un code en C efficace pour que le robot suive les lignes de la piste le plus vite possible et sans faire d'erreurs.

NOTRE ROBOT

Caractéristiques techniques de la base Pololu 3Pi+

Pololu 3Pi+ Diagram
Châssis du robot
Une structure compacte et très légère, idéale pour changer de direction rapidement sur la piste sans perdre de vitesse.
Moteurs et roues
Deux moteurs à courant continu reliés à des roues avec encodeurs pour mesurer et contrôler précisément la vitesse de chaque côté.
Capteurs infrarouges
Une barrette de 5 capteurs de réflectance placée à l'avant pour détecter la position de la ligne noire au sol.
Carte de contrôle
Un microcontrôleur cadencé à 20 MHz qui reçoit les données des capteurs et ajuste la trajectoire des moteurs en temps réel.
Alimentation
Le robot fonctionne avec des piles rechargeables, associées à un régulateur de tension pour garder des performances stables.

Parcours 3 - Détection des tirés (Dashes)

Ce code permet de suivre la ligne noire et d'arrêter le robot après le 4ème tiré transversal. On a ajouté un filtre anti-rebond dans la boucle pour éviter de compter plusieurs fois la même ligne par erreur, et on utilise une régulation proportionnelle simple avec un coefficient Kp de 0.06.

#include <Pololu3piPlus32U4.h>
using namespace Pololu3piPlus32U4;

LineSensors sensors;
Motors motors;

uint16_t s[5];
int baseSpeed = 110;
float Kp = 0.06;
int lastError = 0;

int noLineCount = 0;
const int NO_LINE_THRESHOLD = 25;
const int LINE_DETECT_MIN = 200;

int crossingCount = 0;
const int MAX_CROSSINGS = 4;
bool onDash = false;
bool waitingToStop = false;
int debounceCount = 0;
const int DEBOUNCE_LOOPS = 80;

#define DASH_SENSOR    s[4]
#define DASH_NEIGHBOR  s[3]
const int DASH_THRESHOLD = 500;

void setup()
{
  delay(1000);
  for (int i = 0; i < 80; i++)
  {
    motors.setSpeeds((i < 40) ? 60 : -60, (i < 40) ? -60 : 60);
    sensors.calibrate();
  }
  motors.setSpeeds(0, 0);
}

void loop()
{
  sensors.readCalibrated(s);

  int maxVal = 0;
  for (int i = 0; i < 5; i++)
    if (s[i] > maxVal) maxVal = s[i];

  if (maxVal < LINE_DETECT_MIN)
  {
    noLineCount++;
    if (noLineCount >= NO_LINE_THRESHOLD)
    {
      motors.setSpeeds(0, 0);
      while (1);
    }
  }
  else noLineCount = 0;

  if (debounceCount > 0)
  {
    debounceCount--;
  }
  else
  {
    bool dashAlone = (DASH_SENSOR > DASH_THRESHOLD) && (DASH_NEIGHBOR < DASH_THRESHOLD);

    if (dashAlone && !onDash)
    {
      onDash = true;
      crossingCount++;
      debounceCount = DEBOUNCE_LOOPS;

      if (crossingCount >= MAX_CROSSINGS)
        waitingToStop = true;
    }

    if (!dashAlone && onDash)
    {
      onDash = false;

      if (waitingToStop)
      {
        motors.setSpeeds(0, 0);
        while (1);
      }
    }
  }

  int error = (s[4] + s[3]) - (s[0] + s[1]);
  if (s[2] > (s[1] + s[3]) / 2)
    error = (s[3] - s[1]);
  if (abs(error - lastError) > 1500)
    error = lastError;

  lastError = error;

  int correction = error * Kp;
  int leftSpeed  = baseSpeed + correction;
  int rightSpeed = baseSpeed - correction;
  leftSpeed  = constrain(leftSpeed, -150, 200);
  rightSpeed = constrain(rightSpeed, -150, 200);
  motors.setSpeeds(leftSpeed, rightSpeed);
}

Note sur les tests - Problème de stabilité

« Ça fonctionne une fois sur deux... c'est aléatoire... il n'y a rien qui change dans le code. »
Analyse de l'équipe : Le robot subit parfois des ralentissements bizarres ou perd le fil de la ligne. La calibration des capteurs infrarouges est très sensible aux changements de lumière dans la salle et aux petits défauts du sol. On doit retravailler les seuils de détection et stabiliser le code avant le jour du concours.
